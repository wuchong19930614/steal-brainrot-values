# 03-核心页面与工具簇

## 目标

把网站从单一 value list 页面建设成可以承接多个搜索意图的小工具簇。

核心不是写博客，而是让用户进入后能直接得到答案或使用工具。

- `/`
- `/calculator/`
- `/tier-list/`
- `/rarity-list/`
- `/updates/`

## 首页：Values

路径：

`/`

目标关键词：

- `steal a brainrot values`
- `steal a brainrot value list`
- `steal a brainrot values list`
- `sab values`

页面必须包含：

- H1：`Steal a Brainrot values`
- Last updated
- 可搜索 value table
- rarity filter
- trend filter
- demand filter
- obtainable filter
- sort
- official sources checked
- related tools
- FAQ

当前重点：

- value 为 `TBD` 时，解释为什么。
- 不要让用户误以为这是完整官方价格表。
- 内链要帮助 Google 发现 4 个工具页。
- 首页不要频繁改 title 和 H1，先等 GSC 数据。

验收标准：

- 用户打开后 3 秒内知道这是 value tracker。
- 用户能搜索 item。
- 用户能点击官方 source。
- 用户能进入 calculator / tier list / rarity list / updates。
- GSC 首页 canonical 与线上 canonical 一致。

## Calculator

路径：

`/calculator/`

目标关键词：

- `steal a brainrot calculator`
- `steal a brainrot trade calculator`
- `steal a brainrot trading calculator`

当前状态：

因为官方没有公开 trade value，calculator 应该禁用。

禁用态必须说明：

`Official public sources do not currently publish Steal a Brainrot trade values, so the calculator is disabled until verified values are added.`

未来开启条件：

- 至少 10 个 item 有 verified value。
- value 来源清晰。
- 每个参与 calculator 的 item 都 `value > 0`。
- 页面清楚标注 `Fan-made estimate`。

开启后功能：

- 添加自己的 offer。
- 添加对方 offer。
- 调整数量。
- 删除 item。
- 显示双方 total。
- 显示 win / fair / loss。

验收标准：

- 无 value 时没有 Add 按钮。
- 有 value 时可以完整比较交易。
- 不允许 `TBD` 参与计算。
- 用户操作可通过 GA4 event 追踪。

## Tier List

路径：

`/tier-list/`

目标关键词：

- `steal a brainrot tier list`
- `best brainrot in steal a brainrot`
- `what is the best brainrot in steal a brainrot`

当前状态：

因为没有 verified value，所以 tier list 应显示禁用空态。

未来开启条件：

- 有 verified value。
- 有 demand 判断。
- 有 tier 规则。
- 页面说明 tier 是 fan-made ranking，不是官方排名。

建议 tier 规则：

- S：高 value + 高 demand + 稀缺。
- A：高 value 或高 demand。
- B：中等 value。
- C：低 value 但可交易。
- D：低 value、无需求或不建议交易。

验收标准：

- 无数据时不展示假排名。
- 有数据时每个 tier 解释清楚。
- item 可跳回 value table 或详情页。
- best 相关 query 出现后，再考虑独立 best 页面。

## Rarity List

路径：

`/rarity-list/`

目标关键词：

- `steal a brainrot rarity list`
- `steal a brainrot limited`
- `steal a brainrot secret`
- `what is the rarest brainrot in steal a brainrot`

页面功能：

- 按 rarity 分组。
- 显示每组数量。
- 显示 value range。
- 列出 item。

规则：

- value range 只统计 `value > 0`。
- 没有 verified value 时 range 显示 `TBD`。
- rarity 可以来自官方或人工确认，但必须标记来源。

验收标准：

- 用户能快速知道有哪些 rarity。
- 用户能看到每个 rarity 下有哪些 item。
- 不显示假 value range。
- 后续 rarest 页面可以从这里扩展。

## Updates

路径：

`/updates/`

目标关键词：

- `steal a brainrot value changes`
- `steal a brainrot update values`
- `new brainrots in steal a brainrot`

页面功能：

- 记录数据变更。
- 记录新增 item。
- 记录来源变更。
- 记录 value 变更。

每条 update 应包含：

- date。
- title。
- summary。
- item。
- change type。
- note。

验收标准：

- 用户能知道最近一次数据为什么变。
- 搜索引擎能看到 freshness signal。
- 内部团队能追溯数据来源。
- 每次数据变更都能反查到对应 source。

## 未来页面触发条件

- `/best-brainrots/`

- 核心 5 页至少 3 页已索引。
- Semrush / Trends 显示该词仍在上升。
- 页面能提供真实结构化内容，而不是薄文章。

### Item Detail

路径建议：

`/brainrots/[slug]/`

上线条件：

- item 数量足够。
- 每个 detail 页不薄。
- 至少有 source、aliases、notes、update history、related items。
- 不需要凭空写 300-500 字 filler。

# Targets 文档索引

这个目录管理 `Steal a Brainrot Values` 的 SEO 实验、数据策略、页面扩展、监控复盘和商业化节奏。

所有判断必须基于 MCP 可观测数据（Search Console、GA4、Google Trends、Semrush、线上抓取），不按感觉扩页面。

## 单一数据源原则

每类信息只在一个文件里维护，其他文档一律引用链接，不复制内容：

| 信息 | 唯一归属 |
| --- | --- |
| 商业目标、阶段战略、收入模型 | [00-目标](./00-目标.md) |
| 接入配置（域名、GSC / GA4 / Clarity ID）、当前 P0 清单 | [01-上线与基础配置](./01-上线与基础配置.md) |
| 官方来源清单、数据分级、value 展示规则 | [02-数据策略与可信度](./02-数据策略与可信度.md) |

| 站点数据快照（GSC / GA4 / Clarity / 线上抓取）、记录表 | [04-收录监控与数据看板](./04-收录监控与数据看板.md) |

## 阅读顺序

1. [00-目标](./00-目标.md)：为什么做、做到什么程度。
2. [04-收录监控与数据看板](./04-收录监控与数据看板.md)：当前数据处于什么状态。
3. [01-上线与基础配置](./01-上线与基础配置.md)：当前 P0 要做什么。
4. 其余文档按需查阅。

## 维护规则

- 每次数据复盘（每日 / 每周）：只改 04 的快照和记录表；关键词数据有变化时改 05。
- 改商业目标或收入模型：改 00-目标。
- 改域名、GSC、GA4、Clarity 配置或 P0 清单：改 01。
- 改数据来源、value 展示规则、可信度规则：改 02。
- 改页面模块或工具状态：改 03。
- 新增页面类型或关键词批次：改 05，内容模板规则看 06。
- 改扩展 / 优化 / 停止的判断标准：只改 08。
- 准备广告：先确认 07 的流量门槛。

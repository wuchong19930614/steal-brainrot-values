## Search Console 要看什么

每天看：

- Pages indexing。
- Sitemap status。
- Queries。
- Pages。
- Countries。
- Devices。

最重要指标：

- impressions。
- clicks。
- average position。
- CTR。
- indexed pages。

当前判断优先级：

1. 先看页面是否进入索引。
2. 再看是否有 impressions。
3. 最后才看排名和 CTR。

## GA4 要看什么

重点指标：

- users。
- sessions。
- page views。
- engagement rate。
- average engagement time。
- landing page。
- source / medium。

当前最重要的是：

- 是否出现 organic search。
- 用户是否从首页进入后点击工具页。
- 工具页是否有页面浏览。
- calculator 禁用态是否造成快速离开。

建议后续新增事件：

- `calculator_used`
- `item_filter_used`
- `outbound_official_link_clicked`
- `related_tool_clicked`
- `source_section_clicked`

## Clarity 要看什么

重点看：

- 用户是否滚动。
- 是否点击工具入口。
- 是否在 value table 搜索。
- 是否卡在 calculator disabled state。
- 移动端有没有布局问题。

不要只看 session 数量，要看行为质量。

## 2-4 周观察周期

### 第 1 周

- 确认 sitemap 被读取。
- 确认首页被索引。
- 让 4 个工具页进入 Google 发现队列。
- 不新增页面。

### 第 2 周

- 看是否出现 impressions。
- 看出现的 query 是否和预期一致。
- 看哪个页面先有数据。
- 对 title / intro 只做小改。

### 第 3-4 周

- 判断主词是否进入前 100。
- 判断 calculator / tier list 是否有独立 impressions。
- 记录长尾 query。
- 决定是否发布第二批 3-5 个页面。

## 决策规则

### 情况 A：有 impressions

行动：

- 优先优化已有页面 title 和 intro。
- 根据 query 增加内部链接。
- 扩展 3-5 个最接近的长尾页。

### 情况 B：有收录但没有 impressions

行动：

- 检查页面是否太薄。
- 增强首页和工具页的可用信息。
- 用 Trends / Semrush 选择一个高意图测试页。

### 情况 C：工具页没有收录

行动：

- 用 URL Inspection request indexing。
- 增强首页到工具页的内链。
- 检查 sitemap 是否包含正确 URL。
- 保持 URL 和 canonical 稳定。

### 情况 D：有点击但停留差

行动：

- 优化首屏。
- 强化 `TBD` 原因和 source 解释。
- 减少误导性的工具入口。
- 增加明确下一步链接。

## 验收标准

这一步完成的标准：

- Search Console 已验证。
- sitemap 已提交且无错误。
- 5 个核心页面都被 Google 发现。
- 至少首页被索引。
- GA4 正常采集。
- Clarity 正常采集。
- 每周有固定记录。
- 根据真实数据做扩展决策。

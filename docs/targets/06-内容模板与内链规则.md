# 06-内容模板与内链规则

## 目标

保证每个新增页面都有清晰搜索意图、足够信息量和合理内链，避免生成大量薄页面。

## 页面基础模板

每个页面至少包含：

- 唯一 H1。
- 简短说明。
- Last updated。
- 主体工具或数据模块。
- 官方来源或数据来源。
- 相关页面链接。
- FAQ 或说明区。
- JSON-LD。

页面必须先解决用户问题，再考虑 SEO 文案。

## Title 规则

Title 要包含主要关键词，但不要堆砌。

当前核心 title：

- `Steal a Brainrot Values - Value List, Demand & Trade Prices`
- `Steal a Brainrot Calculator - Check Fair Trades`
- `Steal a Brainrot Tier List - Best Brainrots Ranked`
- `Steal a Brainrot Rarity List - All Brainrots by Rarity`
- `Steal a Brainrot Value Updates - Latest Changes`

新增页面示例：

- `Steal a Brainrot Trading Values - Trade Value Tracker`
- `Best Brainrots in Steal a Brainrot - Ranked by Value Status`
- `All Brainrots in Steal a Brainrot - Rarity and Value Status`
- `Rarest Brainrots in Steal a Brainrot - Limited and Secret List`

避免：

- 太长。
- 过度承诺。
- 写 official 但实际不是官方。
- 写 2026 但没有持续维护计划。
- 堆叠多个近义词。

## Description 规则

Description 要说明页面能解决什么问题。

需要包含：

- 页面功能。
- 数据状态。
- 用户收益。

如果 value 是 TBD，要明确说：

`official trade values are still unpublished or awaiting verification`

不要用 description 暗示数据已经完整或官方确认。

## H1 规则

H1 要简单直接：

- `Steal a Brainrot values`
- `Steal a Brainrot calculator`
- `Steal a Brainrot tier list`
- `Steal a Brainrot trading values`
- `Best Brainrots in Steal a Brainrot`

不要把 H1 写成营销标题。

## 内容质量规则

每个页面必须回答用户的真实问题。

如果用户搜索 value：

- 告诉他是否有 value。
- 告诉他来源。
- 告诉他为什么是 TBD。
- 给他相关 item 或工具。

如果用户搜索 calculator：

- 解释 calculator 状态。
- 有数据时允许计算。
- 没数据时说明原因。

如果用户搜索 tier list / best：

- 有数据时展示排名。
- 没数据时不要编排名。
- 可以展示“当前无法排名的原因”和“哪些数据会用于排名”。

如果用户搜索 rarity / rarest：

- 展示 rarity 分组。
- 说明哪些是官方确认、哪些待确认。
- 不凭空推导稀缺度。

## 内链规则

每个页面至少链接到 2 个相关页面。

基础内链：

- Values -> Calculator
- Values -> Tier List
- Values -> Rarity List
- Values -> Updates
- Calculator -> Values
- Calculator -> Tier List
- Tier List -> Values
- Tier List -> Calculator
- Rarity List -> Values
- Rarity List -> Calculator
- Updates -> Values

未来新增页面内链：

- Trading Values -> Values
- Trading Values -> Calculator
- Best Brainrots -> Tier List
- Best Brainrots -> Values
- All Brainrots -> Rarity List
- All Brainrots -> Values
- Rarest Brainrots -> Rarity List
- Rarest Brainrots -> Updates
- Item detail -> Values
- Item detail -> Calculator
- Item detail -> Same rarity items
- Item detail -> Related aliases

## URL 规则

使用短、清晰、稳定的 URL。

推荐：

- `/calculator/`
- `/tier-list/`
- `/rarity-list/`
- `/updates/`
- `/trading-values/`
- `/best-brainrots/`
- `/all-brainrots/`
- `/rarest-brainrots/`
- `/brainrots/festive-67/`

避免：

- 动态参数 URL。
- 日期无意义堆叠。
- 重复路径。
- 同义页面互相 cannibalize。
- 为同一意图同时创建 value-list 和 values-list。

## FAQ 规则

FAQ 应该回答真实问题。

可以回答：

- Are these values official?
- Why are values TBD?
- How often are values updated?
- Can I use these values for trades?
- What data is needed before the calculator turns on?
- Why is the tier list disabled?

不要写：

- 纯 SEO 堆关键词。
- 没有实际答案的问题。
- 暗示本站是官方资源的问题。

## JSON-LD 规则

推荐：

- `WebSite`：全站。
- `BreadcrumbList`：非首页页面。
- `FAQPage`：只有真实 FAQ 时使用。
- `ItemList`：value table、tier list、rarity list、all brainrots。

不要给没有真实内容的页面堆 schema。

## 发布前检查

新增页面上线前检查：

- Title 唯一。
- Description 唯一。
- H1 唯一。
- canonical 正确且为 www。
- 至少 2 个内链。
- sitemap 包含该页面。
- 没有无来源 value。
- 移动端不溢出。
- 页面不是薄内容。
- build 和 typecheck 通过。

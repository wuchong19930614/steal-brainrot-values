# 05-关键词矩阵扩展

## 目标

把单关键词站点扩展成关键词矩阵，逐步覆盖 500,000+ 月搜索量。

原则：

- 先做低 KD、高意图、低开发成本的词。
- 用 GSC query 决定真正发布顺序。
- 没有数据支撑时，只准备，不批量上线。

## 关键词筛选公式

KDROI：

`KDROI = volume * CPC / KD`

使用方式：

- volume 越高越好。
- CPC 越高越好。
- KD 越低越好。
- KDROI 越高越优先。

但不能只看公式，还要判断：

- 是否是新词。
- 是否有趋势。
- 是否能低成本做出工具或结构化内容。
- 是否有真实用户需求。
- 是否会涉及版权、品牌或官方误导风险。
- 是否有足够数据避免薄页面。

## 页面类型矩阵

### 工具类

优先级最高。

例子：

- calculator。
- value calculator。
- trade checker。
- tier list builder。
- rarity filter。

要求：

- 工具必须真的可用，或清楚说明禁用原因。
- 不允许用 `TBD` 做计算。
- 操作事件应接入 GA4。

### 列表类

例子：

- value list。
- rarity list。
- secret list。
- limited list。
- best brainrots。

要求：

- 结构化。
- 可筛选或可排序。
- source 清楚。
- 内链清楚。

### 单 item 类

例子：

- `[item] value`
- `[item] worth`
- `[item] rarity`
- `[item] in steal a brainrot`

要求：

- 不生成空壳。
- 必须有 source、aliases、notes、related items。
- value 不明确时直接写 `TBD`。

### 更新类

例子：

- latest update values。
- new brainrots。
- value changes。
- event items。

要求：

- 必须真实更新。
- 不能伪造 freshness。
- 每条更新能对应到数据变更或来源变化。

## 不做清单

暂不做：

- 大量 item detail 自动生成。
- 需要实时价格 API 的功能。
- 没有 verified value 的完整排行榜。
- 冒充 official 的标题或页面。
- 纯文章式低信息量页面。

## 验收标准

每批新增页面后都要检查：

- 页面是否被收录。
- 是否有 impressions。
- 是否带来新 query。
- 是否提高站内 PV。
- 是否增加用户停留。
- 是否没有破坏数据可信度。

如果新增页面 4-8 周完全没有 impressions：

- 检查页面质量。
- 合并薄页面。
- 改 title 和内链。
- 或停止该方向。

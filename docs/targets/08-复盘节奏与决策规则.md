# 08-复盘节奏与决策规则

## 目标

建立固定复盘机制，让项目根据真实数据迭代，而不是凭感觉不断加页面。

## 每日复盘

上线后前 7 天每天看一次：

- sitemap indexed 数是否变化。
- 4 个工具页是否从 unknown 进入 discovered / crawled / indexed。
- 首页 user canonical 是否更新为 www。
- GSC 是否出现新的 impressions。
- GA4 是否出现 organic search。
- Clarity 是否出现真实 session。

## 每周复盘

每周固定看一次：

- Search Console impressions。
- Search Console clicks。
- Query 列表。
- Page 列表。
- GA4 users。
- GA4 page views。
- GA4 source / medium。
- Clarity session。
- 新增页面收录情况。
- Semrush URL organic。
- Trends 相关词走势。

每周要回答：

1. 哪些页面有 impressions？
2. 哪些 query 是预期外的？
3. 哪些页面有点击？
4. 用户是否点击工具入口？
5. 有没有页面需要改 title？
6. 有没有值得新增的长尾页面？
7. 是否仍应保持不扩页面？

## 每月复盘

每月看趋势：

- impressions 是否增长。
- clicks 是否增长。
- average position 是否改善。
- 页面数量和流量是否成比例增长。
- 是否有页面完全没有表现。
- 是否有 query 指向未覆盖意图。

每月要做一个决定：

- 继续优化现有页面。
- 新增一批页面。
- 合并薄页面。
- 暂停某个方向。
- 开始或继续不做广告。

## 扩展决策规则

### 可以扩展

满足任意条件：

- 某类 query 连续出现 impressions。
- 某个页面进入前 30。
- 某个页面 CTR 明显高。
- 用户在 Clarity 中频繁点击某类入口。
- Semrush / Trends 显示相关词上升。
- 核心 5 页至少 3 页被发现或索引。

### 暂缓扩展

出现以下情况：

- 现有页面没有收录。
- sitemap 有错误。
- 页面内容太薄。
- 关键词没有 impressions。
- 数据来源不清楚。
- calculator / tier list 仍因无 value 禁用，且用户行为差。

### 停止方向

满足以下情况：

- 8 周没有 impressions。
- 页面无法提供有效内容。
- 需要大量维护但没有流量。
- 可能导致数据误导或品牌风险。

## 页面优化决策

如果 impressions 高但 clicks 低：

- 优化 title。
- 优化 description。
- 更明确匹配 query。
- 增强 FAQ。

如果 clicks 有但停留低：

- 优化首屏。
- 提高工具可用性。
- 减少误导。
- 强化相关链接。

如果排名卡在 20-50：

- 增加内容深度。
- 增加内链。
- 更新 freshness。
- 增加结构化数据。

如果排名进入前 10：

- 不要大改页面。
- 只做小幅优化。
- 增强转化和 PV。
- 观察稳定性。

## 收入决策

如果月 PV < 10,000：

- 不重点做广告。
- 继续扩关键词和页面，但必须看索引和 impressions。

如果月 PV 10,000-100,000：

- 小规模测试广告。
- 观察 RPM。
- 保持工具首屏干净。

如果月 PV > 100,000：

- 系统优化广告布局。
- 分析页面 RPM。

如果月 PV > 300,000：

- 进入收入增长阶段。
- 重点优化广告平台和页面结构。

## 长期复盘目标

每季度判断一次：

- 这个游戏是否还在增长。
- 关键词趋势是否还在。
- 是否要扩展到其他 Roblox 游戏。
- 是否要复制站点模板。
- 是否要做更大的游戏工具站矩阵。

最终目标不是只做一个站，而是验证一套可复制模型。

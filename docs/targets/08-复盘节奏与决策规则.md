# 08-复盘节奏与决策规则

## 定位

本文件是**复盘节奏和全部决策规则的唯一来源**：什么时候看数据、看什么、看到什么信号做什么决定。

- 数据本身、指标口径和记录表：见 [04-收录监控与数据看板](./04-收录监控与数据看板.md)。
- 关键词数据和扩展批次：见 [05-关键词矩阵扩展](./05-关键词矩阵扩展.md)。
- 广告决策的 PV 门槛：见 [07-广告变现计划](./07-广告变现计划.md)。

项目基线即 [04 每日记录表](./04-收录监控与数据看板.md#记录表)的首行（2026-07-05）。当前 5 个核心页面已收录，短期重点从 request indexing 转为观察 impressions / query / 自定义事件是否连续出现。

## 每日复盘

上线后前 7 天每天检查一次：

- 首页 / sitemap / robots 是否 200。
- sitemap errors / warnings 是否仍为 0，indexed 数是否变化。
- 首页 URL Inspection coverage 是否仍是 indexed，user canonical 是否更新为 www。
- 5 个核心页面是否仍保持 indexed。
- Performance 是否出现新的 page 或 query。
- GA4 是否出现非 direct 来源。
- 最小 GA4 工具事件部署后是否出现。
- Clarity 是否有真实 session。

结果记录到 [04 的每日记录表](./04-收录监控与数据看板.md#记录表)。

## 每周复盘

每周固定看一次（指标口径见 04）：

- Search Console impressions / clicks。
- Query 列表。
- Page 列表。
- GA4 users / page views / source medium。
- Clarity session。
- 新增页面收录情况。
- Semrush URL organic。
- Trends 相关词走势（如有变化，更新 [05 的关键词数据](./05-关键词矩阵扩展.md#当前关键词数据)）。

每周要回答：

1. 哪些页面有 impressions？
2. 哪些 query 是预期外的？
3. 哪些页面有点击？
4. 用户是否点击工具入口？
5. 有没有页面需要改 title？
6. 有没有值得新增的长尾页面？
7. 是否仍应保持不扩页面？

结果记录到 [04 的周报表](./04-收录监控与数据看板.md#记录表)。

## 每月复盘

每月看趋势：

- impressions 是否增长。
- clicks 是否增长。
- average position 是否改善。
- 页面数量和流量是否成比例增长。
- 是否有页面完全没有表现。
- 是否有 query 指向未覆盖意图。

每月要做一个决定：

- 继续优化现有页面。
- 新增一批页面。
- 合并薄页面。
- 暂停某个方向。
- 开始或继续不做广告。

## 扩展决策

**这是全套文档中「是否新增页面」的唯一判断标准，其他文档（00 / 01 / 03 / 05）一律引用本节。**

### 硬性门槛（全部满足才能发布新页面）

1. 核心 5 页中至少 3 页已被 Google 索引（不只是被发现）。
2. GSC 连续 7 天出现 impressions。
3. 新页面能提供真实工具或结构化内容，不是薄文章（内容标准见 [06](./06-内容模板与内链规则.md)）。

当前状态：

- 门槛 1 已满足。
- 门槛 2 尚未满足，仍需继续观察。
- 门槛 3 需要在具体页面方案中单独判断。

例外：若门槛 1-2 尚未满足，但 GSC 已连续出现某方向的 query impressions，可提前发布该方向 1 个测试页（门槛 3 仍必须满足），其余方向继续等待。

### 优先级信号（决定先做哪个方向，不构成发布许可）

- GSC 出现 trading / SAB / best / rarest 相关 query。
- Semrush / Trends 显示相关词上升。
- Clarity 中用户频繁点击某类入口。
- 某个页面进入前 30 或 CTR 明显高。

批次划分和各批次附加条件见 [05 的扩展批次](./05-关键词矩阵扩展.md#扩展批次)。

### 暂缓扩展

出现以下情况：

- 现有页面没有收录。
- sitemap 有错误。
- 页面内容太薄。
- 关键词没有 impressions。
- 数据来源不清楚。
- calculator / tier list 仍因无 value 禁用，且用户行为差。

### 停止方向

满足以下情况：

- 8 周没有 impressions。
- 页面无法提供有效内容。
- 需要大量维护但没有流量。
- 可能导致数据误导或品牌风险。

## 页面优化决策

如果工具页长期未被收录：

- 用 URL Inspection request indexing。
- 增强首页到工具页的内链。
- 检查 sitemap 是否包含正确 URL。
- 保持 URL 和 canonical 稳定。

如果有收录但没有 impressions：

- 检查页面是否太薄。
- 增强首页和工具页的可用信息。
- 用 Trends / Semrush 选择一个高意图测试页。

如果 impressions 高但 clicks 低：

- 优化 title。
- 优化 description。
- 更明确匹配 query。
- 增强 FAQ。

如果 clicks 有但停留低：

- 优化首屏。
- 强化 `TBD` 原因和 source 解释。
- 提高工具可用性，减少误导性的工具入口。
- 增加明确下一步链接。

如果排名卡在 20-50：

- 增加内容深度。
- 增加内链。
- 更新 freshness。
- 增加结构化数据。

如果排名进入前 10：

- 不要大改页面。
- 只做小幅优化。
- 增强转化和 PV。
- 观察稳定性。

## 收入决策

广告的开关、布局和各阶段动作全部按 [07 的变现阶段](./07-广告变现计划.md#变现阶段)执行，本文件不重复门槛数字。月 PV 未达到阶段 2 门槛前，复盘时只需确认「继续不做广告」。

## 长期复盘目标

每季度判断一次：

- 这个游戏是否还在增长。
- 关键词趋势是否还在。
- 是否要扩展到其他 Roblox 游戏。
- 是否要复制站点模板。
- 是否要做更大的游戏工具站矩阵。

最终目标不是只做一个站，而是验证一套可复制模型。

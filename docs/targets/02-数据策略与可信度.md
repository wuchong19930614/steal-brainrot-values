# 02-数据策略与可信度

## 目标

建立可信的数据策略，避免把未经确认的社区估值包装成官方数据。

这个站的长期生命力来自信任。宁可显示 `TBD`，也不要展示看起来准确但来源不清的数据。

## 数据分级

### Official

定义：

来自官方 Roblox experience、官方站点、官方公告、官方社媒、官方商品页。

可以展示：

- item 名称。
- item 是否存在。
- item 是否 limited / DLC。
- 官方描述。
- 官方链接。

不能自动推导：

- trade value。
- demand。
- future value。
- 是否值得交易。

### Verified Manual

定义：

人工多来源确认后的维护数据。

进入条件：

- 至少 2 个以上可靠社区来源。
- 来源之间差异可解释。
- 数据更新时间明确。
- 维护者明确知道这是社区估值，不是官方价格。
- updates 页面记录本次加入或变更。

展示方式：

- 标记为 `Fan-made estimate`。
- 有 confidence。
- 有 lastUpdated。
- 有 sourceLabel / sourceUrl。

### Community Estimate

定义：

来自 Discord、Wiki、玩家表格、视频评论、社群截图等来源，但尚未交叉验证。

使用规则：

- 默认不进入核心 value。
- 可以作为内部参考，不直接驱动 calculator。
- 如果展示，必须明显标注不保证准确。
- 不得在 title、H1、FAQ 中写成 official。

### Unknown

定义：

没有可靠来源，或来源无法复查。

展示方式：

- `TBD`。
- 不参与 calculator。
- 不参与 tier ranking。
- 可以出现在 rarity list 和 source tracker。

## 字段要求

每个 item 至少需要：

- `id`
- `name`
- `slug`
- `rarity`
- `value`
- `valueSourceType`
- `demand`
- `trend`
- `tier`
- `obtainable`
- `aliases`
- `notes`
- `confidence`
- `sourceLabel`
- `sourceUrl`
- `lastUpdated`

如果 `value > 0`，还必须补充：

- `valueSourceLabel`
- `valueSourceUrl`
- `valueSourceCheckedAt`

可选补充字段：

- `valueSourceNote`
- `firstSeen`
- `retiredAt`
- `sourceCheckedAt`

## 值展示规则

- `value <= 0` 显示为 `TBD`。
- 只有 `value > 0`、`valueSourceType` 为 `official` 或 `verified-manual`，且 value 来源和核验日期齐全时，才允许进入 calculator 和 tier ranking。
- `sourceUrl` 必须可打开。
- `lastUpdated` 必须跟随数据变更更新。
- `confidence` 只能表达来源可信度，不能表达 value 一定正确。

文案规则：

可以写：

`Official public sources do not currently publish trade values.`

可以写：

`Fan-made estimate, last checked YYYY-MM-DD.`

不要写：

- `official price`
- `confirmed value`
- `guaranteed value`
- `best value`
- `worth 1M`

## 数据更新流程

每次更新数据时按这个流程：

1. 确认来源类型。
2. 保存 source URL。
3. 判断是否满足 Official 或 Verified Manual。
4. 修改 data 文件。
5. 更新全局 `lastUpdated`。
6. 更新 updates 记录。
7. 检查 calculator 和 tier list 是否应开启。
8. 跑 `npm run typecheck` 和 `npm run build`。
9. 上线后观察 GSC、GA4、Clarity 是否有异常。

## Verified Manual 准入标准

一批 value 可以开启 calculator 的最低条件：

- 至少 10 个 item 有 verified value。
- 每个 item 至少 2 个来源。
- 每个 value 有 last checked 日期。
- 有一段页面说明明确这是 fan-made estimate。
- updates 页面说明开启 calculator 的原因。
- 没有 value 的 item 仍然显示 `TBD`，不能混入计算。

## 当前风险

最大风险：

- 为了 SEO 快速填充假数据。
- 用户误以为本站数据是官方价格。
- 社区估值争议导致信任下降。
- calculator 使用了 `TBD` 或低可信数据。

处理原则：

- 短期少一点内容可以接受，错误数据不可以接受。
- 搜索词再诱人，也不能牺牲数据可信度。
- 任何数据变更必须能在 updates 页面追溯。

## 验收标准

这一步完成的标准：

- 所有 item 都有 source。
- 没有无来源 value。
- 无官方或 verified value 的 item 都是 `TBD`。
- calculator 不使用 `TBD` 数据。
- tier list 不使用 `TBD` 数据。
- updates 页面说明数据变化。
- 页面 copy 没有把 fan-made estimate 写成 official。
